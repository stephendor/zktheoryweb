The cocycle is a 1-cochain whose coboundary is zero in the persistence diagram cohomology — its support identifies which edges (pairs of households) carry the persistent cycle. Extract the household indices from these edges and examine their welfare profiles. These are the households whose pairwise proximity at scale b (birth parameter 0.71 in this case) constitutes the geometrical loop. To understand what closes the loop: examine the filtration at scale d (death parameter), find the triangle that fills the cycle, and identify the three households forming that triangle — these are the households whose mutual proximity at scale d triangulates the void. ## The Study You Will Design

Every module in Path 4 has developed one piece of the TDA practitioner toolkit: computing persistence (Modules 1–3), interpreting diagrams (Module 4), testing statistical significance (Module 5), applying Mapper (Module 6), tracking features through time with zigzag (Module 7), handling multi-parameter complexity (Module 8), vectorising for downstream analysis (Module 9), integrating with deep learning (Module 10), and auditing algorithmic fairness (Module 11). This final module asks you to assemble all of it into a coherent study design. Not a tutorial exercise, but a real study — the kind you will actually run on welfare data, submit to a social policy journal, and defend to reviewers who may include both TDA specialists and empirical social scientists with no prior exposure to computational topology.

The checklist below does not provide a recipe. It provides a structured set of decisions that every TDA study must make explicitly, with enough reasoning to explain each choice to a non-specialist and to reviewers in revision. The checklist is organised in the order decisions must be made, from the initial framing of the topological claim through to the final reporting standard.

## 1. Research Question Framing

**What topological claim are you making?**

Every TDA study rests on an implicit or explicit topological claim: the data has a particular global structure — clusters, loops, voids, branching — that is not adequately described by mean values or correlations. Before computing anything, state the claim precisely.

A well-formed topological claim has three parts: (a) a specified homology dimension (H₀ for clustering, H₁ for loops, H₂ for voids); (b) a persistence threshold (the minimum persistence that constitutes a finding rather than noise); and (c) a substantive interpretation that is falsified if the topological structure is absent. Example: "We claim that the joint welfare trajectory data contains a persistent H₁ feature — indicating cyclical poverty dynamics — with persistence exceeding the 95th percentile of the bootstrap null distribution. Absence of this feature would falsify the poverty-trap hypothesis as stated."

Stating the claim up front disciplines the entire analysis. It forces a decision about which homology dimension matters, prevents post-hoc adjustment of the significance threshold, and makes the substantive interpretation testable rather than vague.

## 2. Data Preprocessing Decisions

**Which normalisation, which distance, which subsampling?**

Each of these decisions is a theoretical commitment about what kind of variation matters (Module 2):

- **Normalisation choice**: z-score treats all variables as theoretically commensurable; domain-normalisation (e.g., dividing by median income) preserves economic interpretation. State the choice and justify it in one sentence referencing the research question.
- **Distance metric**: Euclidean is the default; cosine is appropriate when welfare profile type matters more than welfare level; precomputed matrices allow custom metrics. If your distance choice changes the finding, report both results and discuss the discrepancy.
- **Subsampling**: If $n > 5{,}000$, use maxmin landmark subsampling (Module 2). Report the landmark count and the convergence evidence — that increasing landmarks does not substantially change the diagram.

Document these choices in a dedicated "Computational Methods" subsection of your paper. Reviewers from TDA backgrounds will check these first; reviewers from social science backgrounds will check whether the choices are theoretically motivated.

## 3. Algorithm Selection

**VR vs Čech vs alpha complex; standard vs zigzag vs multi-parameter.**

Select the filtration algorithm based on the data structure, not computational convenience:

- **Standard VR (ripser)**: for cross-sectional data or single time-point data. Default choice unless a principled reason exists for alternatives.
- **Alpha complex**: for point clouds in low-dimensional Euclidean space (typically $d \leq 4$) where a geometrically tighter complex is desired. Not appropriate for high-dimensional welfare data.
- **Zigzag**: for longitudinal data where the panel composition changes across waves (Module 7). Required when tracking features through time rather than comparing diagrams.
- **Multi-parameter**: when density and scale are theoretically independent concerns and you need to separate their effects (Module 8). Requires RIVET and substantially more interpretation effort.

Justify the choice in one sentence. If you use standard VR on longitudinal data when zigzag would be more appropriate, that is a methodological limitation to be acknowledged.

## 4. Parameter Sensitivity

**Have you swept the filtration parameter? Tested multiple cover settings?**

No TDA finding is credible if it depends on a single parameter setting. The minimum sensitivity analysis:

- Vary the maximum filtration distance (or the `thresh` parameter in `ripser`) from half to twice the default value. Report whether the primary features are visible across this range.
- If using Mapper: vary resolution from 15 to 40 and gain from 20% to 45%. Report the parameter region where the claimed graph structure (loop, flare, bifurcation) is stable.
- If using landmark subsampling: vary the landmark count from 200 to the minimum feasible full-sample size and report the bottleneck distance between consecutive diagrams as the landmark count increases. A decreasing bottleneck distance demonstrates convergence.

A parameter sweep is not optional. It is the minimum evidence that the finding is a property of the data rather than of a particular algorithmic configuration.

## 5. Statistical Validation

**Do your features survive a bootstrap test?**

Run the bootstrap significance test (Module 5) with $B \geq 1{,}000$ iterations for any H₁ feature you intend to report as a finding. The p-value should be reported with two significant figures (p = 0.031, not p < 0.05). If you are comparing diagrams across groups, run the permutation test (Module 5) as well.

Report the full null distribution — not just the p-value — as a histogram in supplementary materials. This allows reviewers to assess whether the null distribution is well-behaved and whether the observed feature is a clear outlier or a marginal exceedance.

## 6. Vectorisation for Downstream Analysis

**Which vectorisation for which downstream task?**

If the study includes regression, classification, or clustering over multiple diagrams (e.g., comparing persistence across geographic areas or demographic groups), choose and justify the vectorisation method (Module 9):

- **Persistence landscape**: use for interpretable comparison and statistical testing (landscape averages are well-defined).
- **Persistence image**: use for supervised machine learning tasks where compact fixed-dimensional vectors are needed.
- **Wasserstein distance matrix**: use for distance-based clustering (hierarchical clustering, UMAP projection) of diagram collections.

Report the vectorisation hyperparameters (grid resolution, bandwidth, number of landscape levels) and justify their choice.

## 7. Interpretation Guard-Rails

**What can topology claim versus what it cannot?**

TDA establishes the existence and robustness of geometric structure. It does not establish:

- **Causality**: a persistent loop does not identify the mechanism that creates it.
- **Individual membership**: which specific households are in a loop is a post-hoc analysis; the H₁ feature does not come with a pre-attached list of members.
- **Prediction**: the presence of a loop in the current data does not predict how long it will persist or what would dissolve it.
- **Policy prescription**: a topological finding is evidence for a structural argument; the argument must be made with additional tools.

Include a one-paragraph "Interpretation Limits" section in every TDA paper. It demonstrates methodological honesty and pre-empts the most predictable reviewer complaints.

## 8. Reporting Standards

**Include diagrams, parameter choices, null model results, and interpretation guard-rails.**

A complete TDA replication package includes:

1. The persistence diagram(s) as publication-quality figures, with diagonal drawn and significance threshold annotated.
2. All preprocessing parameters (normalisation, distance metric, landmark count, maximum filtration distance).
3. The bootstrap or permutation null distribution as a histogram.
4. The full Mapper graph (if used) with parameter settings documented.
5. Code to reproduce all computations from the preprocessed data matrix.
6. The "Interpretation Limits" paragraph discussed in item 7.

The single most common TDA reporting error is reporting only findings. A credible TDA paper reports what topology was found, what topology was absent, at what parameter settings, against what null model, and with what limitations.

## Path 4 Retrospect

This path has taken you from installation to frontier methodology. The gradient has been: computation (Modules 1–3) → interpretation (Modules 4–5) → advanced tools (Modules 6–9) → applications (Modules 10–11) → design (Module 12). The methods are now a toolkit; the research programme represented by Papers 1–10 is the context in which that toolkit has been deployed and tested. Designing your own study means making explicit choices that, in the papers, appear as settled conventions — but which were each the result of exactly this kind of principled decision procedure, documented here so that the next generation of TDA welfare researchers does not have to reconstruct it from first principles.
