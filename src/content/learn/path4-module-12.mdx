## When the Network Should Know About Loops

Deep learning and topological data analysis have developed largely independently, each producing powerful tools for different kinds of structure. Deep learning excels at learning hierarchical representations from raw data, extracting features that are predictive of task labels without explicit feature engineering. TDA excels at characterising the global geometric structure of data in a way that is stable, scale-free, and theoretically grounded. For welfare research, both capabilities are needed: the data is high-dimensional and complex enough to benefit from deep representation learning, yet the research questions concern structural properties — cyclical poverty dynamics, stratification geometry, boundary structure — that TDA characterises precisely and neural networks can dissolve.

The integration challenge is therefore not "which tool to use" but "how do the tools reinforce each other?" Two integration strategies have emerged as the principled approaches: **topological feature extraction**, which treats TDA as a preprocessing step that augments the feature set before network training; and **topological regularisation**, which adds a topological loss term to the training objective, constraining the network's learned representations to preserve specified topological structure. This module covers both strategies, their implementation in `pytorch-topological`, and their application to poverty-score prediction where the topological structure of welfare trajectories should be preserved rather than collapsed in the latent space.

## Strategy 1: Topological Feature Extraction

In topological feature extraction, TDA is used to derive a fixed-length numerical description of each data point's local or global topological neighbourhood, and that description is appended to the original feature vector before passing to a standard neural network.

The pipeline for a welfare classification task:

1. For each geographic area or individual, compute a neighbourhood subgraph representation (e.g., the $k$-nearest neighbours in the welfare-variable space).
2. Run `ripser` on the subgraph's distance matrix to produce H₀ and H₁ persistence diagrams.
3. Vectorise each diagram using a persistence landscape (Module 9) or persistence image.
4. Concatenate the landscape or image vector with the original feature vector.
5. Train a standard MLP, random forest, or gradient boosting model on the augmented features.

This approach is simple to implement and does not require specialised TDA-aware training infrastructure. Its limitation is that the topological features are precomputed and fixed — the network cannot learn to weight different aspects of the persistence diagram differently during training. The landscape vectorisation is a linear function of the diagram, and if the task-relevant topology is encoded nonlinearly, the extraction step may discard it before training begins.

## Strategy 2: Topological Regularisation

Topological regularisation adds a **topological loss** $\mathcal{L}_{\text{topo}}$ to the training objective, penalising the network when the persistence diagram of its output (or internal representation) deviates from a target topology:

$$\mathcal{L}_{\text{total}} = \mathcal{L}_{\text{task}} + \lambda \cdot \mathcal{L}_{\text{topo}}$$

The topological loss is typically the sum of squared persistence discrepancies between the current diagram and the target diagram, differentiable with respect to the network weights through the use of sub-gradient or implicit differentiation techniques. The `pytorch-topological` library implements several differentiable topological loss functions including the **signature loss** (penalising the difference in persistence barcodes), the **Wasserstein topological loss** (penalising diagram distance from a target), and the **connectivity loss** (penalising unwanted connected components in the output).

## Sublevel Persistence of Neural Activations

A third integration point is **post-hoc audit**: after training a standard neural network on welfare prediction, apply TDA to the network's internal activations to determine whether the learned representation preserves the known topological structure of the data.

The tool for this is sublevel set persistence. For a trained network $f: \mathbb{R}^d \to \mathbb{R}^k$, extract the intermediate-layer activations $a(x) \in \mathbb{R}^m$ for all data points $x$ in the validation set. Construct a scalar function $g: X \to \mathbb{R}$ from these activations (e.g., the norm $\|a(x)\|_2$ or the first PCA component of the activation matrix). Compute the sublevel set persistence of $g$ — the persistent homology of the filtration defined by $g^{-1}(-\infty, t]$ as $t$ grows from $\min g$ to $\max g$. The resulting diagram characterises the topology of the activation landscape.

For a welfare-score predictor where the original data has a known persistent H₁ feature (Module 3), the sublevel persistence of the final hidden layer should also show a persistent H₁ feature of comparable magnitude if the network learned a topologically faithful representation. A flat diagram (no persistent H₁ in the activation landscape) indicates the network learned to solve the task by collapsing the cyclical structure — perhaps exploiting a correlation that is not the structural signal.

## Application: Poverty-Score Prediction with Topological Fidelity

The motivating application from **Papers 7 and 8** is poverty-severity scoring for policy targeting: predict a welfare-vulnerability score for each household that reflects not just current income but the structural trajectory risk implied by the household's position in welfare space. Standard regression models trained to minimise squared prediction error will learn to approximate the mean trajectory, potentially discarding the cyclical structure that predicts recurrence risk. A topological regularisation approach adds a loss term penalising the disappearance of the H₁ features in the latent space, ensuring the learned score function preserves the poverty-cycle structure detected by TDA rather than averaging over it.

The connection to Papers 7 and 8 is direct: Paper 7 develops the sublevel persistence audit methodology for welfare predictors trained on Universal Credit assessment data, finding that standard gradient-boosting models consistently suppress the persistent H₁ feature even when achieving high predictive accuracy. Paper 8 tests whether topological regularisation recovers this structure without sacrificing predictive performance, and evaluates the trade-off between task accuracy and topological fidelity as a function of the regularisation weight $\lambda$.
