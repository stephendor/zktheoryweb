## The Density Confound

Standard Vietoris–Rips persistence has a well-documented limitation when applied to data with heterogeneous density: it conflates scale effects with density effects. Consider a welfare dataset with two sub-populations — a large dense cluster of working-age households near the poverty line and a small sparse cluster of households with chronic severe deprivation. In a single-parameter VR filtration, the dense cluster generates many short-lived H₀ features at small scales (components quickly merging) and perhaps a persistent loop as the cluster forms a rough ring in variable space. The sparse cluster generates long-lived H₀ features (its components merge only at large scales because the households are far apart) and suppresses any H₁ analysis because sparse regions require large scales before triangulation is possible.

The scale parameter $r$ is simultaneously doing two jobs: capturing the geometric scale of features and compensating for the density variation of the point cloud. Increasing $r$ to reach the sparse cluster simultaneously over-triangulates the dense cluster, destroying topological features in the dense region before the sparse region is even connected. The solution — computing separate VR diagrams for each cluster after density-based stratification — is ad hoc and introduces arbitrary thresholding. **Multi-parameter persistence** provides the principled alternative: filter by both $r$ and a density threshold $\rho$ simultaneously, producing a two-dimensional parameter space where scale and density are genuinely independent.

## The Bifiltration

Given a point cloud $X$ and a density estimator $\hat{\rho}: X \to \mathbb{R}$ (the kernel density estimate at each point), the **bifiltration** $K_{(r, \rho)}$ is the simplicial complex that includes all simplices whose vertices are (a) within distance $r$ of each other and (b) have density estimate at least $\rho$:

$$K_{(r, \rho)} \subseteq K_{(r', \rho')} \quad \text{for } (r, \rho) \leq (r', \rho') \text{ component-wise}$$

That is, $r' \geq r$ (scale grows) and $\rho' \leq \rho$ (density threshold is lowered, admitting sparser points). The nesting condition is satisfied: increasing $r$ adds new edges and simplices; decreasing $\rho$ admits new low-density vertices with their incident edges. The result is a two-parameter family of nested complexes — a functor from the poset $\mathbb{R} \times \mathbb{R}^{\text{op}}$ to the category of simplicial complexes.

The **two-parameter persistence module** associated with this bifiltration is the functor $H_k(K_{(r, \rho)})$ assigning to each parameter pair $(r, \rho)$ the $k$-th homology vector space of the corresponding complex, with maps induced by inclusions.

## The Failure of the Structure Theorem

For one-parameter persistence modules over a field $\mathbb{F}$, the structure theorem guarantees a unique bar-code decomposition. This theorem rests on the classification of graded modules over the polynomial ring $\mathbb{F}[t]$, which is a principal ideal domain (PID). Every finitely generated module over a PID decomposes as a direct sum of cyclic modules — the bar-code intervals.

The two-parameter ring $\mathbb{F}[r, \rho]$ is not a PID. Graded modules over $\mathbb{F}[r, \rho]$ do not admit a complete discrete decomposition: the category of finitely presented $\mathbb{F}[r, \rho]$-modules contains indecomposables of arbitrarily complex structure, and there exists no finite combinatorial invariant that fully classifies them. The bar-code, which was the complete invariant for one-parameter persistence, has no direct analogue in the two-parameter setting.

This is not merely a theoretical inconvenience. Applied TDA researchers who expect multi-parameter persistence to produce "a bar-code with two axes" will be disappointed — that object does not exist in the theory, and any tool claiming to produce it is either restricting to a special sub-class of modules or producing an approximation with known error.

## Invariants That Do Exist

Three classes of invariants are practically useful for two-parameter persistence:

**The Hilbert function** $H(r, \rho) = \dim_\mathbb{F} H_k(K_{(r, \rho)})$ assigns to each parameter pair the dimension of the homology vector space — the total count of independent $k$-cycles present at that parameter value. It is a non-negative integer for each $(r, \rho)$ in the discretised parameter grid. A 2D heatmap of the Hilbert function visualises where homological features are present; ridges of elevated values in the heatmap indicate regions of parameter space where the data has consistent loop structure.

**The rank invariant** $\rho(p, q) = \text{rank}(H_k(K_p) \to H_k(K_q))$ for $p \leq q$ in the poset gives the number of $k$-cycles at parameter $p$ that persist to parameter $q$. For a grid of parameter pairs, the rank invariant forms a 4D tensor, but it is typically examined as a family of 2D slices fixing two of the four parameters.

**Fibered bar-codes** decompose the bifiltration into a family of one-parameter restrictions — essentially projecting the 2D parameter space onto one-parameter lines and computing a standard bar-code for each line. The collection of bar-codes across all lines (or a representative sample) characterises the global structure without requiring a single summary.

## RIVET: Interactive Bipersistence Visualisation

The **RIVET** software (Rivet Interprets Virtually Everything in Topology) is the primary interactive tool for two-parameter persistence analysis. RIVET takes a bifiltration as input and computes the minimal free resolution of the two-parameter module, from which the Hilbert function and rank invariant can be queried for any parameter pair in real time.

RIVET's interface displays the bifiltration's 2D parameter space with the Hilbert function overlaid as a heatmap, and a "line query" tool that computes the fibered bar-code for any one-parameter line through the space. Moving the line query interactively reveals how the topological summary changes as you sweep through different combinations of scale emphasis and density threshold — the single most powerful diagnostic for understanding whether a topological feature is genuinely independent of density, or merely a density artefact visible at particular scale–density combinations.

RIVET is available as a standalone executable and as a Python interface via `pyrivet`. For welfare-data applications, the typical workflow is: compute the bifiltration in Python (using a custom distance-matrix and kernel-density-estimate pipeline), export to RIVET's input format, and use RIVET's GUI for interactive exploration before extracting specific numerical invariants for reporting.

## How This Module Connects

Multi-parameter persistence is the methodological innovation at the centre of **Papers 3 and 4**. Paper 3 develops a bifiltration framework for UKHLS welfare panel data that simultaneously filters by normalised welfare distance and by local density (estimated from the kernel density of the welfare variable distribution), providing the first formal test of whether detected poverty-cycle topology is density-independent. Paper 4 extends this to a three-parameter setting (adding survey wave as a temporal parameter), connecting multi-parameter persistence to the zigzag constructions from Module 7. Together, these two papers represent the frontier of applied two-parameter TDA in social science and constitute the principal technical contribution of the research programme to TDA methodology itself.
